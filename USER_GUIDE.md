# 乒乓球拍防伪溯源系统使用手册

欢迎使用基于区块链的乒乓球拍防伪溯源系统。本系统帮助消费者验证球拍真伪，帮助厂商、物流和销售方记录产品的生产与流通过程。

本手册面向**非技术人员**，用通俗易懂的语言介绍系统功能、使用方法和常见问题。

## 系统能做什么

 imagine 你买了一块乒乓球拍，想知道它是正品还是假货。传统方式可能依赖包装、防伪标签或商家信誉，但这些都有可能被仿造。

本系统把每一块球拍的关键信息记录到**百度超级链**上。区块链上的数据具有不可篡改、可追溯的特点，因此：

- **消费者**：输入产品 ID，就能查到这块球拍从生产到销售的完整链路。
- **厂商**：录入自己生产的球拍，生成唯一的产品 ID。
- **物流/经销商**：记录球拍的入库、出库、销售等流转环节。

简言之，系统为每一块球拍建立了一份"数字身份证"，并把它存到了公共可查验的区块链上。

---

## 三种使用角色

| 角色 | 是否需要登录 | 能做什么 |
|------|-------------|----------|
| **消费者** | 不需要 | 在首页输入产品 ID，验证真伪、查看流通记录 |
| **厂商** | 需要 | 创建新产品、查看自己生产的产品、更新产品销售状态 |
| **物流/经销商** | 需要 | 为产品添加物流记录（入库、出库、销售确认） |

系统**不开放普通消费者注册账号**，因为查验真伪只需游客身份即可。

## 消费者：如何验证球拍真伪

### 1. 获取产品 ID

产品 ID 通常印刷在：

- 球拍外包装上的防伪标签
- 球拍手柄处的铭牌
- 厂商提供的电子质保卡或购买凭证

产品 ID 是一串纯数字，例如：`2067626370338394112`。

### 2. 打开验证页面

在浏览器中打开系统首页，例如：

```
http://localhost:5173
```

或部署后的公网地址。

### 3. 输入产品 ID 并验证

1. 在首页输入框中输入产品 ID。
2. 点击"立即验证"按钮。
3. 稍等片刻，系统会显示验证结果。

### 4. 查看验证结果

验证结果分为三种情况：

#### ✅ 验证通过（正品）

页面会显示：

- 产品品牌、型号、材质
- 生产日期
- 当前状态（已生产 / 运输中 / 在库 / 已售出）
- 流通记录：生产 → 物流入库/出库 → 销售确认
- 两项关键指标：
  - **链上已存证**：说明区块链上有该产品的记录。
  - **数据哈希一致**：说明链上记录与系统数据库中的记录匹配，未被篡改。

如果这两项都为"是"，基本可以确认该产品是正品。

#### ❌ 验证失败（可能为假冒）

如果页面提示"未在链上查询到该产品记录"，可能原因：

- 输入的产品 ID 有误（请仔细核对）。
- 该产品从未被厂商录入系统。
- 该产品是假冒产品。

#### ⚠️ 链上记录存在但数据哈希不一致

这种情况较少见，表示：

- 链上有该产品的记录，但系统数据库中的某条记录与链上哈希不匹配。
- 可能数据库被异常修改，或者某次上链操作未完整保存。

遇到此情况，请联系系统管理员或厂商核实。

## 厂商：如何录入产品

### 1. 登录系统

1. 打开系统地址，进入登录页面（通常是 `/login`）。
2. 输入厂商账号和密码。
3. 角色选择"厂商"。
4. 点击登录。

登录成功后，系统会自动跳转到"产品管理"页面。

### 2. 创建新产品

1. 点击"创建产品"或右上角的"新增产品"按钮。
2. 在表单中填写：
   - **品牌**：例如"红双喜"
   - **型号**：例如"狂飙龙5"
   - **材质**：例如"五层纯木"
   - **批次号**：例如"HSX-2026-001"
   - **生产日期**：选择实际生产日期
   - **质检报告哈希**（可选）：如果上传了质检报告，可填写其哈希值
3. 点击"提交"。

系统会：

- 自动生成一个唯一的**产品 ID**（19 位数字）。
- 把产品信息保存到数据库。
- 把关键数据哈希写入区块链。

### 3. 查看已创建的产品

在产品列表页，厂商只能看到自己生产的产品。每行显示：

- 产品 ID
- 品牌/型号
- 当前状态
- 创建时间

点击某一行可以查看详情或更新状态。

### 4. 更新产品销售状态

当产品被售出后，厂商可以将其状态更新为"已售出"：

1. 在产品列表中找到该产品。
2. 点击"更新状态"或进入详情页。
3. 选择新状态（例如"已售出"）。
4. 提交后，状态变更会再次上链存证。

## 物流/经销商：如何记录流转

### 1. 登录系统

与厂商类似，登录时角色选择"物流"或"经销商"。

### 2. 添加物流记录

1. 进入"物流管理"页面。
2. 输入要操作的产品 ID。
3. 选择操作类型：
   - **入库**：产品进入仓库
   - **出库**：产品离开仓库
4. 填写仓库名称和所在位置（城市）。
5. 点击"提交"。

系统会：

- 保存物流记录到数据库。
- 同步把该次流转的哈希写入区块链。
- 自动更新产品状态（例如入库后变为"在库"，出库后变为"运输中"）。

### 3. 查看物流记录

在物流管理页面，可以按产品 ID 查询该产品所有的入库、出库记录。

## 影响校验结果的关键因素

验证结果由多个因素共同决定，了解这些因素有助于正确解读结果。

### 1. 产品 ID 必须正确

产品 ID 是唯一标识。输入错误（多一位、少一位、数字错）都会导致验证失败。

### 2. 链上必须有记录

厂商创建产品时，系统会把数据写入区块链。如果写入失败（例如 gas 不足、网络异常），链上就没有记录，验证会失败。

### 3. 链上链下数据哈希必须一致

系统会对比：

- 链上保存的 `data_hash`
- 系统数据库中保存的 `data_hash`

两者一致，才说明数据未被篡改。如果某次上链后数据库记录被异常修改，就会出现"链上有记录但哈希不一致"的情况。

### 4. 合约账户必须有足够 gas

每次写入区块链都需要消耗 gas（类似手续费）。如果合约账户余额不足，上链会失败。

### 5. 私钥和密码必须正确

系统需要私钥才能调用区块链。如果私钥文件路径错误、密码错误或私钥被更换，后端无法正常写入或查询链上数据。

### 6. 网络必须可达

后端需要能够访问百度超级链开放网络节点（`39.156.69.83:37100`）。如果服务器网络不通，上链和验证都会失败。

## 常见问题排查

### 问题 1：消费者验证时提示"未在链上查询到该产品"

可能原因和解决办法：

| 原因 | 解决办法 |
|------|----------|
| 产品 ID 输入错误 | 仔细核对包装或凭证上的 ID |
| 厂商尚未录入 | 联系厂商确认是否已创建 |
| 上链失败 | 联系管理员检查 gas 余额和网络 |
| 合约配置错误 | 联系管理员检查合约名和节点地址 |

### 问题 2：厂商无法登录

- 确认用户名和密码正确。
- 确认角色选择为"厂商"。
- 如果忘记密码，需要联系系统管理员重置。

### 问题 3：创建产品时提示"上链存证失败"

- 检查后端日志，查看具体错误信息。
- 常见原因：
  - 合约账户 gas 不足，需充值。
  - 私钥密码错误。
  - 网络无法访问超级链节点。
  - 合约未部署或合约名配置错误。

### 问题 4：物流记录添加成功但状态未更新

- 刷新页面后查看状态是否变化。
- 如果状态长时间未变，可能是上链延迟或失败，联系管理员查看日志。

### 问题 5：验证结果显示"数据哈希不一致"

- 这表示链上记录与数据库记录存在差异。
- 不要自行判断产品真假，应立即联系厂商或系统管理员核实。

## 安全提示

### 对消费者

- 请从正规渠道购买产品，并向商家索取产品 ID。
- 不要轻信非官方验证页面，谨防钓鱼网站。
- 验证时请仔细核对网址是否与官方一致。

### 对厂商和物流/经销商

- 账号密码请勿告知他人。
- 离开电脑时请退出登录。
- 私钥文件和密码由系统管理员保管，普通用户无需接触。

### 对系统管理员

- 私钥文件必须存放在安全位置，严禁上传到 Git 或网盘。
- 数据库密码、JWT 密钥应通过环境变量注入，不要写入代码。
- 定期更换 JWT 密钥和数据库密码。
- 生产环境务必使用 HTTPS。

## 术语解释

| 术语 | 通俗解释 |
|------|----------|
| **区块链** | 一种分布式账本技术，数据写入后难以篡改，适合存证。 |
| **产品 ID** | 每一块球拍的唯一编号，类似身份证号。 |
| **哈希（Hash）** | 一种数字指纹。任何数据经过哈希计算都会得到一串固定长度的字符串，数据稍有改动，哈希就会完全不同。 |
| **上链** | 把数据写入区块链，获得不可篡改的存证。 |
| **gas** | 调用区块链合约时需要支付的手续费。 |
| **私钥** | 区块链账户的"钥匙"，拥有私钥才能代表账户发起交易。必须严格保密。 |
| **data_hash** | 产品或物流数据的哈希值，用于校验链上链下数据是否一致。 |
| **tx_hash** | 某一次上链交易的哈希值，可在区块链浏览器中查询该笔交易。 |
